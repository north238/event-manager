# プロジェクト概要

## プロジェクト名

イベント管理ツール

## コンセプト

「ゆるく始められるイベント管理ツール」

URLを共有するだけで参加者を募ることができ、イベント主催者が参加方法（ゲスト参加/SNSログイン/アカウント登録）を選択できる柔軟なイベント管理システム。

## 背景と課題

### 既存サービスの問題点

- **connpass**: 全員がアカウント登録必須 → カジュアルな利用に向かない
- **調整さん**: ゲスト参加のみ → いたずら・なりすましのリスクが高い
- **Peatix**: 有料イベント向け → 無料の小規模イベントには重い

### 解決したい課題

1. カジュアルな友人の集まりから、しっかりした勉強会まで対応したい
2. 参加のハードルを下げつつ、いたずらは防ぎたい
3. イベントの性質に応じて、主催者が参加方法を選べるようにしたい

## ターゲットユーザー

### 主催者

- 友人グループで飲み会・旅行を企画する人
- エンジニア勉強会・座談会を主催する人
- 社内イベント・コミュニティイベントの運営者

### 参加者

- 気軽にイベントに参加したい人
- 会員登録の手間を省きたい人
- 本人確認が必要なイベントに参加したい人

## ユースケース

### ユースケース1: 友人との飲み会

```
主催者: 友人グループのLINEで「飲み会やります！」とURLをシェア
参加者: URLをクリック → 名前とメールアドレスを入力 → 参加完了
主催者: 参加者リストを確認して、お店を予約
```

- **参加方法**: ゲスト参加OK
- **重視する点**: 参加のハードルの低さ
- **許容するリスク**: 多少のいたずらは気にしない

### ユースケース2: エンジニア勉強会

```
主催者: Twitterでイベント告知、URLをシェア
参加者: URLをクリック → GitHubでログイン → 参加完了
主催者: 参加者リストで本人確認、会場のキャパを確認
```

- **参加方法**: SNSログイン必須
- **重視する点**: 本人確認、いたずら防止
- **リスク管理**: 定員管理が必要

### ユースケース3: 社内イベント

```
主催者: 社内チャットでURLをシェア
参加者: URLをクリック → 社内アカウントでログイン → 参加完了
主催者: 参加者の所属部署を確認、備品の準備
```

- **参加方法**: アカウント登録必須
- **重視する点**: 確実な本人確認、参加履歴の記録
- **リスク管理**: 外部の人は参加できない

## コアバリュー

このシステムが提供する本質的な価値：

1. **イベントを作成できる**
   - タイトル、日時、場所、説明を入力するだけ
   - 一意なURLが自動生成される

2. **参加方法を選べる**
   - ゲスト参加OK: 名前とメールのみ
   - SNSログイン必須: Google/GitHub/LINE
   - アカウント登録必須: このサービスのアカウント
   - イベントの性質に応じて主催者が選択

3. **URLを共有して参加者を募れる**
   - LINE、Twitter、Slackなど、どこでも共有可能
   - QRコード生成（将来的に）

4. **参加者を確認・管理できる**
   - リアルタイムで参加者リストを確認
   - 定員管理
   - 参加者の強制削除

## 差別化ポイント

### 既存サービスとの比較

| サービス       | 参加方法           | 対象                 | 差別化ポイント                     |
| -------------- | ------------------ | -------------------- | ---------------------------------- |
| connpass       | アカウント登録必須 | 勉強会・イベント     | 固定的、カジュアルな利用に向かない |
| Peatix         | アカウント登録必須 | 有料イベント中心     | 無料の小規模イベントには重い       |
| 調整さん       | ゲスト参加のみ     | 日程調整             | いたずらリスクが高い               |
| Doorkeeper     | アカウント登録必須 | コミュニティイベント | 固定的、カジュアルな利用に向かない |
| **本システム** | **選択可能**       | **全てのイベント**   | **柔軟性が高い、唯一無二**         |

### 独自の強み

1. **柔軟な参加方法**
   - イベントごとに参加方法を選択できる
   - 主催者がリスクとハードルのバランスを決められる

2. **低いハードル**
   - SNSログインで会員登録不要でも参加可能
   - 「ちょっと参加してみるか」が実現できる

3. **適度な安全性**
   - ゲスト参加でもメールアドレスで一定の本人確認
   - SNSログインで実質的な本人確認
   - いたずら防止とユーザビリティのバランス

## 用語定義

### 主催者（Organizer）

イベントを作成し、管理する人。必ずアカウント登録が必要。

### 参加者（Participant）

イベントに参加登録する人。参加方法はイベントタイプによる。

### ゲスト参加（Guest Participation）

アカウント登録不要で、名前とメールアドレスのみで参加できる方法。

### SNSログイン（Social Login）

Google、GitHub、LINEなどの外部SNSアカウントでログインする方法。会員登録は不要。

### アカウント登録必須（Account Required）

このサービスのアカウント作成が必要な参加方法。

### slug

イベントのURLに使用される一意な文字列。例: `/events/abc12345`

### participation_token

参加者がキャンセルする際に使用する一意なトークン。
